2019.11.06: A brief introduction to provable security

This post is primarily aimed to new students of cryptography who struggle with provable security.

A security reduction is a proof that demonstrates [1][2][3]:

“any encryption scheme, when combined with this other thing in this specific way, results in a system with security property X, as long as the encryption scheme satisfies property Y”

The syntax of these homework problems is typically in the form:

show [Thing-Π is Y secure] ⇒ [Thing-Π is X secure]

Thing-Π is actually a placeholder which can substituted with a real concrete implementation.

Let's look at a concrete example:

  • define Thing-Π as any encryption scheme using a block cipher in counter mode with Encrypt-then-MAC
  • replace (instance) Thing-Π with AES256-CTR-HMAC-SHA256
  • replace Y with IND-CCA2
  • replace X with IND-CPA

We have [AES256-CTR-HMAC-SHA256 is IND-CCA2 secure] ⇒ [AES256-CTR-HMAC-SHA256 is IND-CPA secure].

Note, AES256-CTR-HMAC-SHA256 is an instance of Thing-Π. We could have used something else like CHACHA20-POLY1305.

One can think of [AES256-CTR-HMAC-SHA256 is IND-CCA2 secure] as an advertisement. The "product" AES256-CTR-HMAC-SHA256 has all the security features of IND-CCA2 security. Caveat emptor, provable security isn't a silver bullet on security claims, as with all matters in life, it helps to maintain a healthy level of skepticism. Of course, this is all theoretical, in practice nobody really advertises their product relies on IND-CCA2 security.

  • the security of Thing-Π (which is Y-secure) is X-secure
  • the security of Thing-Π with Y-security reduces to X-security
  • Thing-Π Y-security is at least as secure as X-security
  • a generic attack on security property X means an attack on any scheme satisfyin Y-security.
For example, if someone says their encryption scheme is IND-CCA2 secure then you can assume it resists all attacks to IND-CPA (a weaker security notion).
